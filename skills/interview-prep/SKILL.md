# /interview-prep — 面试准备（支持一面 & 后续面）

面试前一站式准备：公司调研、JD拆解、定制自我介绍、预判问题与答案、反问清单。
**自动检测**：如果该公司已有 prep.md，自动询问是否切换到后续面准备模式。

## 用法

```
/interview-prep [公司名] [岗位名]
```

启动后如果没有提供公司名/岗位名/JD，主动询问用户。

## 配置文件

启动时**必须**先读取配置文件 `~/.claude/skills/interview-config.md`。

**如果配置文件不存在或未填写**，在执行任何步骤前，引导用户完成配置：
1. 询问用户的简历存放目录
2. 询问面试准备文档的存放目录
3. 询问目标岗位和偏好方向
4. 询问经历特点（如"经历跨度大需要主动说明联系"，或留空）
5. 将信息写入 `~/.claude/skills/interview-config.md`

配置文件中的路径和信息将在下文中以 `{简历目录}`、`{面试准备目录}`、`{目标岗位}`、`{偏好方向}`、`{经历特点}` 表示。

## 自动加载

启动时自动读取以下文件（用户无需手动提供）：

1. **简历**：读取 `{简历目录}` 下最新的 PDF 文件
2. **面试共性问题复盘**：读取 `{面试准备目录}/面试共性问题复盘.md`（如有）

## 模式检测逻辑

启动后，用 Glob 扫描公司文件夹 `{面试准备目录}/*{公司名}*/`：

- **没有 prep*.md** → **一面模式**（走下面的"执行流程"）
- **有 prep*.md** → 用 AskUserQuestion 问用户："检测到已有面试准备文档，这是后续面准备吗？"
  - **否** → 覆盖重做（一面模式）
  - **是** → **后续面模式**（跳到下面的"后续面模式执行流程"）

## 一面模式执行流程

### Step 1: 读取简历和背景
- 用 Glob 找到 `{简历目录}/*.pdf`，读取日期最新的那份
- 读取 `面试共性问题复盘.md`（如有）

### Step 2: 确认信息
- 确认公司名、岗位名
- 如果用户还没贴 JD，请用户提供岗位 JD

### Step 3: 公司调研（用 WebSearch）

调研分为**通用模块**（所有公司都查）和**岗位相关模块**（根据 JD 定制）。

#### 3a. 通用模块（不随岗位变化）

**初创/小厂：**
- 公司主体：国内/海外实体、法人、股东结构（天眼查/企查查）
- 创始人背景：学历、履历、此前创业经历
- 融资轮次、投资方、估值、ARR 量级
- 团队规模和构成（技术/运营/产品占比）
- 近期新闻、媒体报道

**大厂/知名企业：**
- 该业务线/部门在集团内的定位和战略优先级
- 近期组织调整、人事变动、业务方向变化
- 该岗位所在团队的规模和汇报线

**所有公司都查：**
- App Store / Google Play 评分和近期评价（如有产品）
- 社交媒体存在感（公众号、小红书、Discord、Twitter 等）

#### 3b. 岗位相关模块（根据 JD 定制）

**产品分析（当 JD 涉及产品运营/产品经理时）：**
- 核心产品定位、目标用户、商业模式
- 产品所处阶段（冷启动/增长期/成熟期/转型期）
- 自己下载体验产品，记录一手使用感受和改进点
- 用户评价中的高频好评和差评

**竞品分析（当 JD 涉及产品/策略/市场时）：**
- 同赛道 2-3 个主要竞品
- 与竞品的差异化优势和劣势
- 该公司在行业中的位置

**内容/社区生态分析（当 JD 涉及内容运营/社区运营时）：**
- 内容形态、分发渠道、内容调性
- 社区活跃度和官方运营现状
- 创作者/用户生态现状

**技术/工具分析（当 JD 涉及 AIGC/技术相关时）：**
- 核心技术栈和技术壁垒
- AI 工具矩阵及使用场景
- 同类工具对比

**海外市场分析（当 JD 涉及海外/出海/国际化时）：**
- 目标市场（国家/地区）及文化特征
- 本地化策略和挑战
- 海外营销节点日历（节日、文化事件）
- 目标市场用户画像和内容消费偏好

### Step 4: 拆解岗位 JD
- 提炼 3-5 个核心能力要求
- 逐一与简历经历匹配，用表格展示：

| 能力要求 | 匹配经历 | 匹配度 | 说明 |
|---------|---------|--------|------|

- 标出匹配度高的（强项）和匹配度低的（需要桥接的）

### Step 5: 生成定制自我介绍
- 基于 JD 最重要的 3 个能力点，结构化打出"我有这三点"
- 30秒版 + 1分钟版
- **如果 JD 或公司涉及英语（海外产品、要求英语能力、国际化团队等），同时生成中英双语版本**
- 要求：
  - 每个能力点必须有具体事例+数据佐证，不能只说"我自认为有"
  - 如果用户在配置文件中标注了经历特点（如"经历跨度大"），在自我介绍中主动说明经历之间的联系
  - 结尾桥接到目标岗位："这些经验和贵司的XX场景是相通的"
  - 英文版避免 filler words（like, some kind of），结构清晰（背景→现在→优势→连接岗位），控制在 1.5 分钟内

### Step 6: 预判面试问题 + 答案大纲
分类列出可能被问到的问题：

**经历类：**
- 基于简历每段经历的深挖问题
- 每个问题给 STAR 答案框架（用真实经历+数据）
- 标注"容易被追问的地方"和追问的应对

**动机类：**
- 为什么选这个公司/岗位
- 职业规划
- 离职原因（只讲"我想要什么"，不讲"现在有什么不好"）

**能力/假设类：**
- 基于 JD 的假设情景题
- 团队协作、项目管理类问题

**关键要求：**
- 每个答案都要有 solid 案例，不能空谈
- JD 中匹配度低的能力点，准备"虽然没有 A，但我有 BCD"的桥接话术
- 每个回答结尾强制加一句桥接句，关联到目标岗位
- **如果 JD 或公司涉及英语，为核心问题（自我介绍、项目经历、岗位理解、优劣势）准备中英双语答案大纲**

### Step 7: 反问清单
生成 5-8 个可以问面试官的问题：
- 关于岗位日常工作和团队
- 关于业务方向和挑战
- 展示你对公司/行业的研究深度（"我了解到贵司……我的理解是……请问是否准确？"）

### Step 8: 保存文档
将完整准备材料保存到：
```
{面试准备目录}/{MMDD} {公司名}/prep.md
```
日期格式：MMDD（如 0323），取当天日期。

## 后续面模式执行流程

当用户确认是后续面准备时，执行以下流程。

### Step F1: 读取所有已有材料
- 读取该公司文件夹下的所有 prep*.md（一面/历次准备）
- 读取该公司文件夹下的所有 review/复盘相关文件（如 `*复盘*`、`*review*`）
- 读取该公司文件夹下的录音稿（如 `*录音稿*`）
- 读取简历（同一面模式）和 `面试共性问题复盘.md`（如有）

### Step F2: 收集上一轮信息

**如果没有找到 review/复盘文件**，请用户提供：
- 上一轮录音稿或问答回忆（基于此做一次快速复盘，提取关键信息）
- 面试官透露的关键信息（团队规模、产品方向、用户画像等）

**如果找到了 review/复盘文件**，直接读取后，再问用户：
- 下一轮面试官是谁？（HR / 业务负责人 / CEO / 技术 / 其他）
- 已知的考察重点（如果知道的话）
- 有没有需要补充的面试官透露信息

### Step F3: 差距分析（后续面独有）

生成三个分析板块：

**1. JD 能力要求 × 已考察情况**

用表格标注每项能力点的考察状态：

| JD 能力要求 | 上轮是否考察 | 考察情况/表现 | 下轮策略 |
|------------|-------------|-------------|---------|

- 已考察且表现好 → 标注"保持"
- 已考察但表现一般 → 标注"需改进"，给出改进方向
- 未考察 → 标注"预判下轮考察"，预判对应问题

**2. 上轮薄弱点**

从 review/复盘中提取表现 B/C 档的问题，逐一标注：
- 问题是什么
- 当时回答的问题在哪
- 下轮如何改进（改进版答案要点）

**3. 面试官透露信息汇总**

整理成结构化情报表：

| 信息 | 来源 | 下轮如何使用 |
|------|------|------------|

### Step F4: 更新自我介绍
- 基于上轮反馈调整侧重点（如一面重内容能力，二面换产品思维/管理视角）
- 融入上轮学到的信息（如面试官透露的用户画像、团队情况）
- 修正上轮暴露的表达问题（如英文 filler words、结构不清等）
- **如涉及英语，生成中英双语版本**
- 30秒版 + 1分钟版（如有需要，加3分钟版）

### Step F5: 精进已有问题 + 补充新问题

**已有问题升级（基于 review/复盘中的评级）：**
- **A 档（过关）**：保持原答案，标注"继续用"
- **B 档（需修改）**：给出改进版答案大纲，标注改了什么、为什么改
- **C 档（需重构）**：全新答案大纲，说明重构逻辑

**新增问题预判：**
- 基于 Step F3 中未考察的 JD 能力点，预判新问题
- 基于下轮面试官角色预判：
  - CEO → 战略视野、公司理解、长期价值
  - HR → 动机、稳定性、团队 fit、薪酬期望
  - 业务负责人 → 执行力、具体方案、实操能力
  - 技术 → 技术理解、工具使用、数据分析
- **压力面问题**：基于上轮暴露的弱点设计针对性压力题 + 应对策略

每个新增问题都要给出完整答案大纲（同一面模式的标准：STAR法/具体案例+数据+桥接句）。
**如涉及英语，核心问题提供中英双语答案大纲。**

### Step F6: 更新反问清单
- 基于面试官透露的信息设计更深层的反问（不重复一面已问过的）
- 结合下轮面试官角色调整反问方向：
  - CEO → 公司愿景、战略方向
  - HR → 团队文化、成长路径
  - 业务负责人 → 日常工作、近期目标
- 生成 5-8 个反问，标注每个反问的目的

### Step F7: 生成面试前 Checklist
- 上轮教训的应用清单（从 review 中提取，转化为行动项）
- 需要背熟的答案清单（自我介绍、核心优点、被反驳应对等）
- 行动项（如需要体验产品、准备 demo、研究某个话题等）
- 心态提醒

### Step F8: 保存文档

保存到公司文件夹下，文件名带轮次标记：
```
{面试准备目录}/{公司文件夹}/prep-{轮次}.md
```

轮次命名规则：
- 检测已有 prep 文件数量，自动递增：prep-二面.md、prep-三面.md、prep-终面.md
- 如果用户指定了轮次名称，用用户指定的

**不覆盖原 prep.md**，保留完整的准备历史。

## 内嵌经验教训

以下是面试中常见的血泪教训，在生成所有内容时必须遵循（一面模式和后续面模式通用）：

1. **为每个岗位定制自我介绍**——绝对不要用通用版，要基于 JD 重新组织
2. **任何能力点都要有具体事例+数据佐证**——"我擅长XX"没有说服力，"我做了XX，结果是XX"才有
3. **如果经历跨度大，主动说明联系**——不要等面试官质疑，先讲清楚经历之间的递进逻辑
4. **桥接意识**——每个回答结尾扣回目标岗位，帮面试官理解你的能力是可迁移的
5. **准备了就要说出来**——遇到开放题（"对岗位有什么了解"、"为什么选我们"）要大胆倾倒准备
6. **离职原因只讲"我想要什么"**——不讲现在有什么不好，不吐槽前公司
7. **"虽然没有A，但我有BCD"**——对 JD 中你没有直接经验的要求，准备好桥接话术
8. **反问环节是安全网**——正式回答中忘了展示的准备，用反问的形式补回来
