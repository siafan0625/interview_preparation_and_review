# /interview-review — 面试后复盘

面试后系统复盘：逐题分析、对比准备vs实际、生成改进方案、可选模拟练习。

## 用法

```
/interview-review [公司名] [岗位名]
```

启动后请用户提供面试录音稿或回忆的问答内容。

## 配置文件

启动时**必须**先读取配置文件 `~/.claude/skills/interview-config.md`。

**如果配置文件不存在或未填写**，在执行任何步骤前，引导用户完成配置（流程同 interview-prep）。

配置文件中的路径和信息将在下文中以 `{简历目录}`、`{面试准备目录}`、`{目标岗位}`、`{偏好方向}`、`{经历特点}` 表示。

## 自动加载

启动时自动读取以下文件：

1. **面试前准备文档**：根据公司名在 `{面试准备目录}` 下查找对应文件夹中的 `prep.md`
2. **简历**：`{简历目录}` 下最新的 PDF
3. **面试共性问题复盘**：`{面试准备目录}/面试共性问题复盘.md`（如有）

## 执行流程

### Step 1: 定位面试前准备文档
- 根据公司名在 `{面试准备目录}` 下用 Glob 模糊匹配文件夹
- 读取对应的 `prep.md`
- 如果找不到，告知用户并继续（没有 prep 也可以做复盘）

### Step 2: 获取面试内容
- 请用户提供录音稿或面试问答回忆
- 读取 prep.md 和简历作为对照

### Step 3: 逐题分析
对每一个面试问题，按以下结构分析：

| 维度 | 内容 |
|------|------|
| 面试官问了什么 | 还原问题 |
| 我答了什么 | 还原回答要点 |
| 答得好不好 | 评价（好/一般/差） |
| 为什么 | 具体分析原因 |
| 改进版回答 | 如果答得不好，给出更好的回答框架 |

### Step 4: 按质量分级评估每个回答

将所有回答按答题质量分为三档：

**A. 比较过关的答案 ✅**
- 回答结构清晰、有案例有数据、桥接到位
- 标注亮点：哪些地方特别好，可以沉淀为可复用模式
- 如果有 prep.md，对比是否按准备的框架答的，还是临场发挥更好
- 即使过关，也标注可以微调的地方（更好的措辞、更精准的数据等）

**B. 需要修改的答案 ⚠️**
- 方向对但执行有问题：案例不够 solid、数据缺失、桥接忘了、表达啰嗦、被追问时卡壳
- 对比 prep.md：如果准备了但没答好，分析脱节原因（紧张？问法不同没反应过来？）
- 给出修改版：在原回答基础上补强，标注具体改了哪里、为什么
- **引导用户参与**：问用户"这个问题你当时是怎么想的？有没有其他案例想用？"，结合用户补充的素材优化答案

**C. 需要重构的答案 ❌**
- 方向就有问题：答非所问、逻辑混乱、完全没展示相关能力、踩了雷区（吐槽前公司、空谈没案例等）
- 分析根因：是没准备到？是理解错了问题？还是知识/经验盲区？
- 从头重构：不是修改原回答，而是重新设计回答框架和案例选择
- **引导用户参与**：问用户"这个方向你有没有相关经历可以用？或者你觉得可以怎么答？"，先听用户思路再给建议
- 用 STAR 法则生成完整的新版回答，标注关键数据和桥接句

**额外标注：**
- prep.md 中准备了但面试中完全没机会说的亮点，单独列出，设计主动带出的话术

### Step 5: 生成改进方案

基于 Step 4 的分级，汇总输出：

**修改清单（B 档）：**
- 每个问题列出：原回答要点 → 改了什么 → 改进版回答
- 用户如果补充了案例或思路，融入改进版

**重构清单（C 档）：**
- 每个问题列出：原回答问题诊断 → 新回答框架 → 完整 STAR 回答
- 标注关键数据和桥接句

**可复用模式（A 档）：**
- 沉淀表现好的回答结构，标注适用于哪类问题

**共性问题更新建议：**
- 如果发现了新的共性问题模式，建议更新 `面试共性问题复盘.md`

### Step 6: 保存文档
将复盘报告保存到同一公司文件夹下：
```
{面试准备目录}/{对应公司文件夹}/review.md
```

### Step 7: 可选——模拟提问练习
保存完成后，询问用户：
> 要不要针对薄弱点做一轮模拟练习？我可以扮演面试官，针对你答得不好的地方提问，帮你练习。

如果用户同意：
- 扮演面试官，逐一针对薄弱问题提问
- 用户回答后给出即时反馈（哪里好、哪里可以改进）
- 可以反复练习同一个问题直到用户满意

## 复盘标准

遵循以下原则进行评价：

1. **有没有用具体案例+数据**——空谈能力 vs 有 STAR 回答，差距巨大
2. **有没有桥接到目标岗位**——回答结尾是否关联到了岗位需求
3. **离职原因的表达**——是"我想要什么"还是"现在哪里不好"
4. **经历的递进叙事**——有没有主动说明经历的逻辑联系（如果经历跨度大的话）
5. **准备的深度展示**——开放题中有没有充分展示对公司/岗位的研究
6. **桥接话术**——JD 中没有直接经验的部分，有没有用"虽然没有A，但我有BCD"
